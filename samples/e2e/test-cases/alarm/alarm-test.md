# 预警管理功能测试

**优先级**: P1
**模块**: 预警管理
**前置条件**:
- 应用服务已启动（后端 http://localhost:8080，前端 http://localhost:3000）
- 使用管理员（admin/123456）或安保人员（security001/123456）登录
- 数据库有预警测试数据
- Redis 服务正常（用于 SSE 推送）
**预计耗时**: 180秒

---

## 用例1: 查看预警列表
**编号**: `alarm_list_get_normal_1`

### 步骤
1. 使用 admin/123456 登录系统
2. 导航到预警管理页面
3. 等待预警列表加载（2秒）
4. 查看列表展示

### 验证
- 页面显示预警列表表格
- 表格包含列：预警编号、预警类型、区域、级别、状态、时间等
- 列表显示分页组件
- 网络请求 GET /api/alarm/list 返回 200

### 清理
- 无需清理

## 用例8: 处理预警-正常流程
**编号**: `alarm_handle_normal_1`

### 步骤
1. 在预警管理页面找到一条"未处理"状态的预警
2. 点击"处理"按钮
3. 在处理弹窗中选择处理结果：已确认
4. 填写处理说明："经核实为误报"
5. 点击"确认"按钮
6. 等待处理完成（2秒）

### 验证
- 显示"处理成功"提示
- 预警状态变为"已处理"
- 列表中该预警显示更新后的状态
- 网络请求 POST /api/alarm/{id}/handle 返回 200

### 清理
- 无需清理（保留处理记录）
