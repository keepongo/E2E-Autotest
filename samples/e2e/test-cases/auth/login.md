# 登录功能测试

**优先级**: P0
**模块**: 认证
**前置条件**:
- 应用服务已启动（后端 http://localhost:8080，前端 http://localhost:3000）
- 数据库有测试账号：
  - admin / 123456（管理员）
  - security001 / 123456（安保人员）
  - buyer001 / 123456（买家）
  - seller001 / 123456（卖家）
- Redis 服务正常
**预计耗时**: 120秒

---

## 用例1: 管理员正常登录
**编号**: `auth_login_normal_admin_1`

### 步骤
1. 导航到登录页面 http://localhost:3000/login
2. 等待页面加载完成（2秒）
3. 填写用户名=admin
4. 填写密码=123456
5. 点击"登录"按钮
6. 等待页面跳转（3秒）

### 验证
- URL 跳转到首页或仪表板（/dashboard 或 /home）
- 页面显示用户名"admin"
- 页面显示管理员相关菜单（用户管理、系统设置等）
- 本地存储包含有效的 JWT Token

### 清理
- 点击"退出登录"按钮
- 确认退出操作
- 验证跳转回登录页面

## 用例9: 账号被踢出（SSO 单点登录）
**编号**: `auth_login_sso_kicked_1`

### 步骤
1. 在浏览器 A 中导航到登录页面
2. 在浏览器 A 中使用 admin/123456 登录
3. 验证浏览器 A 登录成功
4. 在浏览器 B 中导航到登录页面
5. 在浏览器 B 中使用 admin/123456 登录
6. 验证浏览器 B 登录成功
7. 切换回浏览器 A
8. 在浏览器 A 中执行任意需要认证的操作（如点击用户列表）
9. 等待响应（2秒）

### 验证
- 浏览器 B 登录成功，可以正常使用
- 浏览器 A 显示提示"您的账号已在其他地方登录"
- 浏览器 A 自动跳转到登录页面
- 截图保存为 sso-kicked-notification.png

### 清理
- 在浏览器 B 中点击"退出登录"
- 关闭两个浏览器窗口

## 用例12: 登录失败次数限制
**编号**: `auth_login_rate_limit_1`

### 步骤
1. 导航到登录页面 http://localhost:3000/login
2. 连续 5 次使用错误密码尝试登录：
   - 填写用户名=admin，密码=wrong1，点击登录
   - 填写用户名=admin，密码=wrong2，点击登录
   - 填写用户名=admin，密码=wrong3，点击登录
   - 填写用户名=admin，密码=wrong4，点击登录
   - 填写用户名=admin，密码=wrong5，点击登录
3. 观察第 5 次后的响应

### 验证
- 前几次显示"用户名或密码错误"
- 第 5 次后可能显示"登录尝试次数过多，请稍后再试"或类似提示
- 或者账号被临时锁定

### 清理
- 等待限制时间过后重新测试正常登录
- 或使用管理员解锁账号
